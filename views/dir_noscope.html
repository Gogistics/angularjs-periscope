On this page we have a controller that contains a directive that does not create a new scope.
  1. Click the #rb#, then click the #psb# to see the initial state. Because the directive does not create a new scope, there is only the controller scope.
  2. Type something in the textboxes, then click the #psb#. Again, since there is only one scope, primitives and object properties associated with |ng-model| simply affect the one scope.

Tip: if your directive does not create its own scope, make sure it does not create new |$scope| properties and polute the parent.
See also this StackOverflow question about how to choose the correct type of scope to use when creating a directive.
